# Security Checklists

**Purpose:** Security, compliance, and vulnerability verification checklists ensuring application and infrastructure safety.

**What Belongs Here:**
- Security design review checklist
- Vulnerability scanning checklist
- Penetration testing checklist
- Authentication and authorization checklist
- Data protection and encryption checklist
- Dependency security checklist
- Infrastructure security checklist
- Compliance verification checklist (GDPR, SOC2, etc.)

**What Does NOT Belong Here:**
- Pre-launch checklists (see ../pre_launch/)
- Code review checklists (see ../code_review/)
- Process checklists (see ../process/)
- Security policies or guidelines (see knowledge/)

**Expected Structure:**
Targeted `.md` checklist files for each security verification area.
