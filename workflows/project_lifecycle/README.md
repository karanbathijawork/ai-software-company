# workflows/project_lifecycle/

**Purpose:** End-to-end workflows defining how projects move from inception through launch and ongoing support.

**Files in This Category:**
- [project-intake-protocol.md](project-intake-protocol.md) — Protocol for orchestrator to turn single user input into structured intake package
- [single_input_product_workflow.md](single_input_product_workflow.md) — Complete workflow allowing single user input to become working MVP without additional clarification

**What Does NOT Belong Here:**
- Individual feature workflows (see ../feature_development/)
- Incident response workflows (see ../incident_management/)
- Cross-functional handoff patterns (see ../cross_functional/)
- Agent definitions (see agents/)

**Expected Structure:**
High-level workflow `.md` files describing agent roles, handoffs, and stage sequences.
