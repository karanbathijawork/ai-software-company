# Feature Development

**Purpose:** Workflows for developing and delivering individual features from design through testing and review.

**What Belongs Here:**
- Feature discovery and requirements workflow
- Architecture and design workflow
- Implementation and development workflow
- Code review workflow
- Testing and QA workflow
- Feature integration workflow
- Release and deployment workflow

**What Does NOT Belong Here:**
- Project-level workflows (see ../project_lifecycle/)
- Incident response workflows (see ../incident_management/)
- Cross-functional patterns (see ../cross_functional/)
- Feature templates (see templates/)

**Expected Structure:**
Detailed workflow `.md` files for each feature development phase.
