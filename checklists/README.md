# checklists/

**Purpose:** Role-specific and lifecycle-specific quality assurance checklists. Used by agents to verify deliverables before completion and by teams to ensure consistency and compliance.

**What Belongs Here:**
- Pre-launch and go-live deployment checklists
- Code review and quality gate verification steps
- Security, compliance, and vulnerability verification
- Process checklists (intake, handoff, closure, retrospectives)

**What Does NOT Belong Here:**
- Agent definitions (see agents/)
- Workflow orchestrations (see workflows/)
- Knowledge or best practices (see knowledge/)
- Templates (see templates/)
- Implementation examples (see examples/)

**Structure:**
```
checklists/
├── pre_launch/    Deployment readiness and go-live verification
├── code_review/   Quality gates, standards, coding practices
├── security/      Compliance, vulnerabilities, safeguards
└── process/       Intake, handoff, closure, retrospectives
```
