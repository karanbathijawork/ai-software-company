# Common Pitfalls

Avoid these mistakes when using this framework:

- Building too much too early
- Trying to solve every possible use case in the first version
- Adding features before validating the core user flow
- Overcomplicating the implementation stack
- Asking too many questions instead of making reasonable assumptions
- Treating the templates as rigid documents instead of starting points

## Rule of thumb

If a feature does not directly support the core user outcome, postpone it.
