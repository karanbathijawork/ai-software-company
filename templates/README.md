# templates/

**Purpose:** Reusable output templates and boilerplate structures that agents use to generate consistent, well-formatted deliverables across different projects and teams.

**What Belongs Here:**
- Technical specification and architecture design templates
- Product documentation templates (PRDs, feature specs, requirements)
- Project management templates (roadmaps, status reports, plans)
- Code artifact templates and boilerplate generators

**What Does NOT Belong Here:**
- Filled-in examples or actual project documents (see examples/)
- Agent definitions (see agents/)
- Workflows or orchestration (see workflows/)
- Checklists or verification steps (see checklists/)
- Implementation code

**Structure:**
```
templates/
├── technical_specs/       System design, architecture, API specs
├── product_documents/     PRD, feature specs, requirements docs
├── project_management/    Roadmaps, status reports, timelines
└── code_artifacts/        Boilerplate, generators, scaffolding
```
