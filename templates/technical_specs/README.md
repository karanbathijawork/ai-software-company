# Technical Specifications

**Purpose:** Templates for technical documents created during system design and architecture phases.

**What Belongs Here:**
- System architecture specification template
- API specification template
- Database schema specification template
- Component design template
- Technology evaluation template
- Risk assessment template
- Performance and scalability specification template

**What Does NOT Belong Here:**
- Filled-in specifications (see examples/)
- Product requirements (see product_documents/)
- Project plans (see project_management/)
- Implementation code

**Expected Structure:**
Template `.md` files with clear sections and examples (not filled in) for each specification type.
