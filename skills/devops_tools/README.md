# DevOps Tools

**Purpose:** DevOps, infrastructure, and deployment tool expertise enabling agents to automate, deploy, and monitor systems.

**What Belongs Here:**
- Docker containerization expertise
- Kubernetes orchestration expertise
- Terraform infrastructure-as-code expertise
- CI/CD platform expertise (Jenkins, GitHub Actions, GitLab CI)
- Monitoring and logging tools (Prometheus, ELK, Datadog)
- Configuration management tools

**What Does NOT Belong Here:**
- Programming language skills (see ../programming_languages/)
- Cloud platform-specific knowledge (see ../cloud_platforms/)
- Generic infrastructure patterns (see knowledge/)
- Implementation scripts (see scripts/)

**Expected Structure:**
One `.md` file per tool category with architecture concepts, best practices, and common patterns.
