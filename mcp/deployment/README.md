# Deployment

**Purpose:** MCP integrations for deployment and CI/CD platforms, enabling agents to trigger builds, deploy applications, and manage releases.

**What Belongs Here:**
- GitHub Actions integration and workflows
- GitLab CI integration and pipelines
- Jenkins integration and job definitions
- AWS deployment service integrations
- Kubernetes deployment integrations
- Container registry integrations (Docker Hub, ECR, etc.)

**What Does NOT Belong Here:**
- Deployment strategies and best practices (see knowledge/)
- Infrastructure definitions (see skills/devops_tools/)
- CI/CD pipeline design patterns (see knowledge/)
- Actual deployment scripts (see scripts/)

**Expected Structure:**
MCP configuration files for each deployment platform and orchestration tool.
