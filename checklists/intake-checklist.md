# Intake Checklist for New Product Requests

Use this checklist before the orchestrator hands off to the Product Manager.

## 1. Clarify the Core Request
- What is the user trying to build or solve?
- Is the request specific enough to define an MVP?
- Is there a clear target user or audience?

## 2. Determine Whether Clarification Is Needed
Ask a clarifying question only if one of these is true:
- the target user is unclear
- the core problem is ambiguous
- the requested outcome is too broad to scope safely
- the requested product type is unclear and affects implementation

If none of these are true, proceed without asking.

## 3. Create a Short Intake Summary
Write a 5-part summary:
- Product idea
- Target user
- Main problem
- Expected outcome
- MVP direction

## 4. Decide the Initial Scope
- Choose the simplest valuable version
- Avoid feature creep
- Prefer a working slice over a large vision

## 5. Hand Off to the Product Manager
Pass the intake summary with this instruction:
- Turn this into a product brief, MVP definition, and prioritized user stories

## 6. Escalate Only if Necessary
Escalate to a human if:
- the user request conflicts with business or technical constraints
- the request is too vague to safely interpret
- the requested product would require major legal, compliance, or security decisions
