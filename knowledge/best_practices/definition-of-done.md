# Definition of Done

A feature or milestone is considered done when all of the following are true:

- The intended user value is clear
- The core user flow works without major issues
- The main acceptance criteria are met
- The implementation is reasonably tested
- The relevant documentation is updated
- The result is ready for review or handoff

## Release-level definition of done

Before considering the MVP complete, confirm that:
- the core workflow is usable
- the app can be started or deployed
- the main success metric can be evaluated
- known limitations are documented
