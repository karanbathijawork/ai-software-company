# Code Review Checklists

**Purpose:** Quality gates and verification steps applied during code review to maintain standards and catch issues early.

**What Belongs Here:**
- Code style and formatting checklist
- Architecture and design review checklist
- Performance and scalability checklist
- Testing coverage checklist
- Documentation completeness checklist
- Dependency and vulnerability checklist
- Accessibility checklist
- Internationalization checklist

**What Does NOT Belong Here:**
- Pre-launch checklists (see ../pre_launch/)
- Security checklists (see ../security/)
- Process checklists (see ../process/)
- Specification templates

**Expected Structure:**
Targeted `.md` checklist files for each code quality dimension.
