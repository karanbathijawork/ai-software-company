# agents/engineering/

**Purpose:** Agent definitions for technical implementation roles across all engineering disciplines.

**Agents in This Category:**
- [Backend Engineer](backend_engineer.md) — Builds reliable APIs, business logic, databases, and integrations
- [Frontend Engineer](frontend_engineer.md) — Builds UI/UX with modern web frameworks and accessibility focus
- [DevOps Engineer](devops_engineer.md) — Builds CI/CD pipelines and cloud infrastructure
- [Software Engineer](software_engineer.md) — Implements features and owns code quality
- [Documentation Engineer](documentation_engineer.md) — Produces user and developer documentation
- [Automation Agent](automation_agent.md) — Eliminates toil through scripting, CI/CD tooling, and automation

**What Does NOT Belong Here:**
- Core leadership roles (see core/)
- Quality and security roles (see quality_assurance/)
- Project or operational coordination (see operations/)

**Expected Structure:**
One `.md` file per engineering discipline, following the 9-section template.
