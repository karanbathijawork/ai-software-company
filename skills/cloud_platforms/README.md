# Cloud Platforms

**Purpose:** Cloud platform expertise enabling agents to design, deploy, and manage infrastructure and services.

**What Belongs Here:**
- AWS expertise (services, best practices, security)
- Azure expertise (services, best practices, security)
- Google Cloud expertise (services, best practices, security)
- Kubernetes expertise (orchestration, deployment)
- Other cloud platforms or serverless platforms

**What Does NOT Belong Here:**
- Programming language skills (see ../programming_languages/)
- DevOps tooling (see ../devops_tools/)
- Generic infrastructure patterns (see knowledge/)
- Project-specific configurations

**Expected Structure:**
One `.md` file per cloud platform with service catalog, best practices, and architectural patterns.
