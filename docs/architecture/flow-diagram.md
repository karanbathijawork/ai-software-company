# Workflow Flow

This repository follows a simple end-to-end flow:

1. User provides a product idea
2. Orchestrator reviews the request and clarifies only when needed
3. Product Manager defines the MVP and product scope
4. Design and architecture work begin from that definition
5. Engineering builds the first working version
6. QA and launch checks validate readiness
7. The product is released and iterated on

## Mental model

Think of the repository as:
- roles for specialization
- workflows for coordination
- templates for speed
- checklists for quality
