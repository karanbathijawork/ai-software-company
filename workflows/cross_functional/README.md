# Cross-Functional Workflows

**Purpose:** Multi-team and multi-role handoff patterns and approval workflows crossing traditional boundaries.

**What Belongs Here:**
- Design approval and sign-off workflow
- Architecture review workflow
- Security review and approval workflow
- Vendor or third-party integration workflow
- Executive sponsorship and sign-off workflow
- Customer feedback integration workflow
- Cross-team dependency management workflow

**What Does NOT Belong Here:**
- Linear project phases (see ../project_lifecycle/)
- Feature-level workflows (see ../feature_development/)
- Incident response workflows (see ../incident_management/)
- Specific checklists (see checklists/)

**Expected Structure:**
Workflow `.md` files describing cross-functional touchpoints and coordination patterns.
