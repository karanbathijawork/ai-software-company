# Decision Logs

**Purpose:** Architectural decisions and technical tradeoffs that explain why specific choices were made and their implications.

**What Belongs Here:**
- Technology selection decisions
- Architectural patterns and justifications
- API design decisions
- Framework and dependency choices
- Infrastructure technology decisions
- Scaling and performance tradeoffs

**What Does NOT Belong Here:**
- General best practices (see best_practices/)
- Project-specific decisions
- Case study narratives (see case_studies/)
- Implementation guidance (see technical_reference/)

**Expected Structure:**
Decision records (ADR format or similar) describing decision context, options, rationale, and consequences.
