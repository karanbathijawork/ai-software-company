# Incident Management

**Purpose:** Workflows for detecting, investigating, resolving, and learning from production incidents and issues.

**What Belongs Here:**
- Incident detection and alerting workflow
- Incident response and escalation workflow
- Root cause analysis workflow
- Resolution and recovery workflow
- Post-incident retrospective workflow
- Incident documentation workflow

**What Does NOT Belong Here:**
- Project lifecycle workflows (see ../project_lifecycle/)
- Feature development workflows (see ../feature_development/)
- Cross-functional patterns (see ../cross_functional/)
- On-call policies or schedules

**Expected Structure:**
Incident phase workflows `.md` files describing agent responsibilities and handoffs.
