# docs/

**Purpose:** Supporting documentation for framework users, contributors, and operators. Includes architecture, usage guides, API reference, troubleshooting, and contribution guidelines.

**What Belongs Here:**
- Framework architecture and design patterns
- User guides and how-to documentation
- API reference for agent interfaces and protocols
- Troubleshooting and debugging guidance
- Contributing guidelines for extending the framework

**What Does NOT Belong Here:**
- Agent definitions (see agents/)
- Knowledge base or case studies (see knowledge/)
- Workflow orchestrations (see workflows/)
- Checklists (see checklists/)
- Implementation examples (see examples/)

**Structure:**
```
docs/
├── architecture/      System design and patterns
├── user_guides/       How to use agents and workflows
├── api_reference/     Agent interfaces and protocols
├── troubleshooting/   Common issues and debugging
└── contributing/      Guidelines for extending framework
```
