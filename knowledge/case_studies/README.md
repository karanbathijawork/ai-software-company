# Case Studies

**Purpose:** Real or realistic examples of projects, decisions, and outcomes that teach frameworks, patterns, and lessons learned.

**What Belongs Here:**
- Post-mortems from successful projects
- Post-mortems from challenging projects
- Feature implementation case studies
- Architecture evolution stories
- Team scaling and organizational lessons

**What Does NOT Belong Here:**
- General best practices (see best_practices/)
- Individual architectural decisions (see decision_logs/)
- Technical reference material (see technical_reference/)
- Project-specific artifacts

**Expected Structure:**
Narrative `.md` files describing context, decision, execution, and outcomes for each case.
