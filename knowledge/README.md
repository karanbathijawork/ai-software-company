# knowledge/

**Purpose:** Shared domain knowledge and reference materials that inform agent decision-making and reasoning across the framework. Not agent-specific; applies broadly to projects and teams.

**What Belongs Here:**
- Best practices and patterns applicable across multiple domains
- Architectural and technical decision logs
- Case studies and lessons learned from real projects
- Technical reference materials, standards, and conventions
- Reusable reasoning frameworks and analysis patterns

**What Does NOT Belong Here:**
- Agent definitions (see agents/)
- Project-specific implementation details
- Individual checklists or verification steps (see checklists/)
- Code examples or templates (see templates/)
- Workflow orchestrations (see workflows/)

**Structure:**
```
knowledge/
├── best_practices/       Patterns, guidelines, and shared wisdom
├── case_studies/         Real-world examples and lessons learned
├── decision_logs/        Architectural decisions and tradeoffs
└── technical_reference/  APIs, standards, frameworks, conventions
```
