# Prompting Best Practices

Use these principles when working with Claude Code through this framework:

- Give one clear goal at a time
- Provide the relevant template or workflow file as context
- Ask for a concrete output, not a general discussion
- Prefer structured deliverables such as PRDs, plans, and checklists
- Tell the system to optimize for the smallest useful MVP
- Ask for assumptions to be stated explicitly

## Recommended prompt pattern

Use a prompt like this:

“Use the product brief in [file] as the source of truth. Create an MVP plan, define the core user flow, and produce implementation-ready tasks.”
